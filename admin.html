Node Management

Rename nodes and assign them to groups. Names appear on the main dashboard.

Nodes are the physical sensor units deployed in the field. You can give each node a display name (e.g. "Purok Riverside") and optionally assign it to a group for shared threshold control. Changes are saved locally and reflected on the dashboard immediately.

Threshold Configuration

Set tilt angle thresholds per node or group. Changes are pushed to ThingSpeak config channels and applied to hardware on the next ESP32 wake cycle (~30 seconds).

L1 = Barangay Officials notified  |  L2 = + BDRRMC  |  L3 = + Purok Residents  |  L4 = All groups — Evacuation order (requires both nodes at L3)

Group Management

Groups allow you to control multiple nodes together. Nodes in the same group share thresholds — changing one updates all. A node can only belong to one group.

Config & API Keys

Register ThingSpeak channel IDs and API keys for each node. These are needed to push threshold changes from this dashboard to the hardware.

Each node needs a data channel (where sensor readings go) and a config channel (where threshold values are written by this admin panel and read by the ESP32). Create config channels in ThingSpeak and enter their details here.
